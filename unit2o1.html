1.5 Describing copyright and other constraints affecting websites

Copyright is rights that the creator of a work is entitled to control, if the work is the users original work of authorship and not someone else's work.
This means that in order to be entitled to copyright protection, the work has to be something you created and not copied from someone else.
This has to be set down in some physical form e.g. like in writing,
in a computer program or on a computer screen.
These rights include:
Distributing the work
Reproduce (or make copies of) the work
Displaying the work e.g. in a public place
Create derivative Works based upon the original work
Performing the work in a public place

1.6 Describing access issues that need taking into account -
describe issues that people with disabilities may face when using webpages and how we can overcome them

There are many issues that may arise when someone with a disability uses a webpage.
Navigating the website may be difficult due to them not being able to read or even see the webpage.
So it will make accessing the website extremely difficult. There are a number of tools available to help with this.
The magnifying glass is an excellent tool as it allows people with low vision to be able to read much easier.
For someone who is blind there is also a screen reader which helps them to easily navigate the page, by reading the webpage out loud.
Another issue can be repetitive strain injury.
Which can be painful to use a mouse and to type for long periods of time.
Websites that function well with a keyboard only (no mouse) work great and vice versa.
But there can be problems when websites can’t be navigated by keyboard commands alone.
People that are deaf may have trouble with video websites like youtube.
Video and other media content are of no use if they don’t contain captions.
When media is captioned, audio is converted to text so it's readable.
Transcripts (written versions of the entire audio or video content) are helpful to.
This allows the user to print the transcripts and use them to study or viewthem on the go from there mobile phone.

1.7 Describe which are the appropriate filetypes for websites

There are many different file types used for websites. The main two being HTML and CSS.
Without these two our website wouldn't work and we wouldn't be able to design it how we want. These are essential files.
For images the most common and widely used type are .jpeg and .png.
.jpeg is great because most photos are in this format. So it makes it great for websites like instagram and facbook.
.png is great because they are usually slightly smaller, and use features like alpha-channel transparency and 24-bit colour support.
This great for adding images into a website.
.zip Zipped files are really groups of other types of files kept together and compressed. This makes it easier to download multiple files all in one go.
.PDF documents can be transferred easily between computers and other platforms.
This great as you can use that file without having to start your own from the beginning.
