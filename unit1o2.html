2.1 Review the ongoing use of IT tools and change approach as needed –
in making your own website, are you happy with the tools you used?

The tools I used in my website project was, visual studio code, trello and google search.
Using Visual Studio Code, is a coding software which allows me to create different types of projects.
It allowed me to code my website how I wanted it to look and what content I wanted to use.
It does take some learning but once you have the basic's down you can start to learn more advanced coding.
This was good for me because I got to learn how HTML and CSS work as it allowed me to learn new ways of coding and even gave me ideas of how I wanted the final look to be.
With the help of some research I found myself getting better at using it and being able to spot errors more easily.
Trello is a free website that is easy to use. It allowed me to keep track of what and where I was in my project.
Looking at code you can easily become lost on what to do next. So Trello allowed me to stay on track and keep pushing forward in completing my website in good time.
Research is always important as it allows me to see how a certain piece of code may work or give me inspiration for my website design.

2.2 Describe whether the IT tools selected were appropriate

Research was a really helpful tool because it gave me access to different ideas, designs and code I may have never have thought of.
This was helpful for me because I didn't know too much about HTML and CSS coding.
I didn't know how to add a navbar or make it move as the user scroll down the page, but some research allowed me to find a use for that code in my website.
Trello kept me on track a number of times. At times I would sit there looking at my code thinking what to do next.
So I would open up Trello and see what tasks I needed to do etc. Or add any new tasks like creating a scrollable nav menu.
Visual Code allowed me to code out my website. This meant I had full control of what and how I wanted it to look.
This gave me the freedom to try different things and experiment with it to see what would look good or not.
Without this program I wouldn't have learnt how to code a basic website nor would I have been able to design it the way I liked.

2.3 Assess the strengths and weaknesses of your final project

Strengths    
Navigating the website is easy and straightforward and it works well.    
Text is easy to read with the background image. It's clear, big and easy to read.    
The borders allow me/the user to easily separate each section of text so it's easy to read and navigate through.    
Headers are also easy, clear and have some good spacing in between each section.   
It's simplistic and isn't difficult to use. Which makes it good for a lot of users.    
Weaknesses    
As I mentioned, simplistic is good, but it's also bad  as it doesn't have much to keep a user interested in staying on the site for long.    
The background is a little stretched in some pages so it doesn't look the same on every page.   
Lack of pictures and colours. Not much to keep the user interested as  I chose mostly white and black colour palette.    
Needs something on the left and right side. It's a little empty there and doesn't look too good.    
Not much on the contact me page. Could of added a little more personality so it.
